## Overview

As part of the ASCTE 401/402 curriculum, our team designed a distributed weather monitoring system. My role was the main software lead, responsible for leading our software group, which was made of two other students, and integrating it into the provided hardware.
## Goals

- Read temperature, humidity, and barometric pressure from physical sensors
- Format and transmit data to a central system
- Design a frontend website to display realtime and historical weather data collected by our system
- Design a secure backend to connect the frontend and the sensor nodes, ensuring data integrity and security
- Handle edge cases gracefully


## Tools & Skills Used

- **Embedded programming** — reading data from physical sensors
- **Data formatting** — structuring sensor output for system-wide use
- **Systems thinking** — designing one component to fit into a larger architecture
- **Web development** — creating a frontend to visualize data
- **Backend development** — building a secure API to connect frontend and sensor nodes
- **Collaboration** — working with a team to integrate different components of the system
- **Git** and **GitHub** — version control and code collaboration

## What I Learned

Leading the software side of the weather station taught me how to integrate embedded systems with a secure backend and a responsive frontend. I learned the importance of handling real-world data carefully, accounting for edge cases like missing readings or network delays. Collaborating with a small team strengthened my ability to coordinate code, debug effectively, and ensure all components fit together smoothly.
